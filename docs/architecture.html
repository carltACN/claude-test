Factory Inventory Management System

System architecture, tech stack, and data flow

Tech Stack
Frontend

Vue 3 + Vite

  • Composition API + script setup
  • Vue Router (hash-based)
  • Axios for HTTP requests
  • Custom SVG charts (no chart lib)
  • i18n: English / Japanese
  • Port: 3000
Backend

Python FastAPI

  • Pydantic v2 data models
  • CORS enabled (all origins)
  • In-memory data (no DB)
  • Auto Swagger docs at /docs
  • uv for dependency management
  • Port: 8001
Data

JSON Files

  • inventory.json — stock items
  • orders.json — 5000+ orders
  • demand_forecasts.json
  • backlog_items.json
  • spending.json — cost data
  • transactions.json
Request / Data Flow
User Interaction

FilterBar.vue

User selects Time Period, Location, Category, or Order Status from 4 dropdowns

↓
State Management

useFilters() composable

Singleton reactive refs shared across all views. getCurrentFilters() returns the current filter object.

↓ watch() triggers reload
HTTP Client

api.js (Axios)

Converts filter object to URLSearchParams. Omits 'all' values. Sends GET requests to FastAPI backend.

↓ HTTP GET localhost:8001
Backend — Route

FastAPI endpoint

Receives query params: warehouse, category, status, month

Backend — Filter

apply_filters()

Sequential in-memory filter: warehouse → category → status → month/quarter

Backend — Validate

Pydantic models

Response schema validated (InventoryItem, Order, etc.) before JSON serialization

↓ JSON response
Component State

Reactive refs

orders.value = data — Vue re-renders template reactively

Derived Data

computed() properties

Secondary statistics (sums, counts, ratios) derived from raw refs. Cached until dependencies change.

Global Filter System

Time Period

?month=
  • Single month: 2025-01…12
  • Quarter: Q1-2025…Q4-2025
  • Orders only (not inventory)

Location

?warehouse=
  • San Francisco
  • London
  • Tokyo
  • Orders + Inventory

Category

?category=
  • Circuit Boards
  • Sensors / Actuators
  • Controllers
  • Power Supplies

Order Status

?status=
  • Delivered / Shipped
  • Processing
  • Backordered
  • Orders only

Composables (Shared Logic)

useFilters()
Singleton filter state. Shared across all views. getCurrentFilters() builds API params.
useI18n()
Translation system for EN/JA. Currency conversion USD ↔ JPY. t() helper for templates.
useAuth()
Mock user authentication state. Provides currentUser and task management.
